# Projects

Use this folder for active projects and initiatives.

## What belongs here

- one folder per active outcome
- intent, status, notes, and supporting material
- child scopes that belong under a project or initiative

## What does not belong here

- ongoing responsibilities that belong in `../areas/`
- shared support material that belongs in `../resources/`
- personal doctrine that belongs in `../policies/`
- historical material that should move to `./vault/`

## Starting a project

Create:

- `README.md` for human navigation
- `INTENT.md` for purpose, end state, constraints, tradeoffs, and revalidation
- optional `notes/`, `resources/`, and `reviews/` folders as needed

Then create a matching Things project with only the next actions.
