# Things Setup

Best-practices reference: [`things-best-practices.md`](./things-best-practices.md)
AppleScript access reference: [`things-applescript.md`](./things-applescript.md)

## Areas

Create these Things Areas:

- Workloop
- Consulting
- Family
- Home
- Personal Ops
- Health
- Relationships
- Money
- Systems
- Creative / Play

## Default Tags

- `urgent`
- `important`
- `deep-work`
- `admin`
- `waiting`
- `call`
- `client`
- `errand`
- `low-energy`
- `medium-energy`
- `high-energy`
- `5m`
- `15m`
- `30m`
- `60m`

Do not add source/category tags such as `email`, `billing`, `stripe`,
`newsletter`, or `family` unless they change how work is chosen. Areas,
projects, and notes should carry that context.

## Project Rule

A Things project should exist only when there is a clear outcome and at least two
next actions. Otherwise, use a single task in the relevant Area.

## Task Rule

Tasks should start with a verb and be doable without rereading the entire
project folder.

Good:

- Draft KWR demo follow-up email
- Review Taxo invoice hours for May 1-7
- Decide next Workloop private alpha blocker

Weak:

- KWR
- Taxo
- Workloop
